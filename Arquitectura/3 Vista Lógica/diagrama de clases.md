# Diagrama de Clases

## ¿Qué es un diagrama de clases?

Un diagrama de clases es un diagrama estático de UML que representa las clases del sistema, sus atributos, métodos y las relaciones entre ellas (herencia, asociación, agregación, composición y dependencias). Se utiliza para modelar la estructura de datos y la arquitectura orientada a objetos, ayudando a diseñar y documentar las responsabilidades de cada clase y cómo se conectan.

Usos comunes:
- Diseño del modelo de dominio y entidades.
- Comunicación entre desarrolladores y arquitectos.
- Base para la implementación y generación de código (cuando aplica).

Buenas prácticas:
- Mantener claridad: modela un subconjunto coherente del sistema por diagrama.
- Usar nombres significativos para clases, atributos y métodos.
- Indicar la visibilidad (+ público, - privado, # protegido) cuando sea relevante.
- Agrupar clases en paquetes o módulos si el diagrama crece.
